# Frontend Checklist - Overview

Dokumen ini berisi turunan dari [Alur Bisnis](../../alur/README.md) menjadi kebutuhan teknis di sisi client (frontend). Checklist ini berfungsi sebagai panduan implementasi UI/UX dan state management.

## Ketentuan Umum
- Checklist frontend adalah turunan dari docs/alur.
- Checklist frontend belum berarti UI harus langsung dibuat.
- Checklist ini menjadi panduan saat implementasi client dimulai.
- Frontend checklist harus menjawab:
  - **Pages / Routes**: Halaman dan alamat URL yang dibutuhkan.
  - **Components**: Komponen UI utama yang perlu dibangun.
  - **User Actions**: Tombol, form, dan interaksi yang tersedia.
  - **UI States**: Penanganan kondisi (Empty, Loading, Error, Success, Revision, Locked).
  - **Role Visibility**: Elemen mana yang tampil/sembunyi berdasarkan role.
  - **Data Display**: Informasi apa saja yang harus ditampilkan.
  - **Form & Validation UI**: Feedback input kepada pengguna.
  - **Batas Fase**: Apa yang tidak dikerjakan di fase ini.
